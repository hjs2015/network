## 第二部分：前言 ⭐

### 为什么需要速算？

在实际工作中，我们经常遇到这样的场景：

| 场景       | 传统方法       | 块计算方法        |
| -------- | ---------- | ------------ |
| **面试现场** | 掏出纸笔算半天    | 30 秒口算出答案 ✅  |
| **会议讨论** | "等我算一下..." | 脱口而出，专业感拉满 ✅ |
| **故障排查** | 打开计算器慢慢算   | 心算定位问题 ✅     |
| **方案设计** | 查表 + 验证    | 快速规划子网 ✅     |

**结论**: 块计算方法 > 二进制转换 > 查表法

### 什么是块计算方法？

**核心思想**: 把复杂的二进制运算，简化为简单的减法运算！

```
【传统方法】
IP: 192.168.1.100
掩码：255.255.255.192
→ 转二进制 → AND 运算 → 转十进制 → 结果
（耗时：2-3 分钟，容易出错）

【块计算方法】
掩码最后段：192
块大小：256 - 192 = 64
网络地址：64 的倍数且≤100 → 64
→ 结果：192.168.1.64
（耗时：10-15 秒，准确率高）
```

**重点核心**⭐: 记住 `256 - 掩码 = 块大小`，其他都是浮云！

## 第三部分：基础概念 📚

### 3.1 子网掩码是什么？

子网掩码（Subnet Mask）用于区分 IP 地址中的**网络部分**和**主机部分**。

```
IP 地址：   192.168.1.100
子网掩码：  255.255.255.0
           ─────┬──── ────┬───
           网络部分   主机部分
```

**通俗理解**: 
- 掩码 255 的段 → 网络部分（保持不变）
- 掩码 0 的段 → 主机部分（可以变化）

### 3.2 CIDR 表示法

CIDR（无类别域间路由）用 `/数字` 表示网络位数。

```
192.168.1.100/24
                ┬
                网络位数（前 24 位是网络部分）
```

**常见 CIDR**:

| 写法      | 含义                 | 子网掩码               |
| ------- | ------------------ | ------------------ |
| /8      | 前 8 位是网络位          | 255.0.0.0          |
| /16     | 前 16 位是网络位         | 255.255.0.0        |
| /24     | 前 24 位是网络位         | 255.255.255.0      |
| /26     | 前 26 位是网络位         | 255.255.255.192    |

### 3.3 关键概念速记

| 概念       | 定义             | 重要性         |
| -------- | -------------- | ----------- |
| **网络地址** | 主机位全为 0 的地址    | ⭐⭐⭐ 子网的标识   |
| **广播地址** | 主机位全为 1 的地址    | ⭐⭐⭐ 子网内广播   |
| **可用主机** | 网络地址和广播地址之间的地址 | ⭐⭐⭐ 实际分配给设备 |
| **块大小**  | 256 - 掩码最后段    | ⭐⭐⭐⭐⭐ 速算核心  |

## 第四部分：理论知识 🔬

### 4.1 块计算法核心公式

```
【核心公式】
块大小 = 256 - 掩码最后段

【推导公式】
网络地址 = 块大小的倍数且≤IP 最后段
广播地址 = 网络地址 + 块大小 - 1
第一个可用 = 网络地址 + 1
最后一个可用 = 广播地址 - 1
可用主机数 = 块大小 - 2
```

### 4.3 网络地址速算技巧 ⭐⭐⭐⭐⭐

**关键步骤**：如何快速找到"块大小的倍数且≤IP 最后段"？

**方法 1：列举法**（适合初学者）
```
例如：IP=200，块大小=32

32 × 1 = 32
32 × 2 = 64
32 × 3 = 96
32 × 4 = 128
32 × 5 = 160
32 × 6 = 192  ← 这个≤200 ✓
32 × 7 = 224  ✗ 超过 200 了

答案：网络地址最后段 = 192
```

**方法 2：除法速算**（推荐！）⭐
```
例如：IP=200，块大小=32

200 ÷ 32 = 6.25
         ↓
       取整数 6
         ↓
       32 × 6 = 192

答案：网络地址最后段 = 192
```

**方法 3：减法逼近**（心算用）
```
例如：IP=200，块大小=32

200 - 32 = 168
168 - 32 = 136
136 - 32 = 104
104 - 32 = 72
72  - 32 = 40
40  - 32 = 8   ← 还能再减
8   - 32 = -24 ✗ 不能减了

所以：200 - (32×6) = 8（余数）
网络地址 = 200 - 8 = 192
```

**实战演练**：

| IP 最后段 | 块大小 | 计算过程 | 网络地址 |
|---------|-------|---------|---------|
| 100     | 64    | 100÷64=1.56→1→64×1 | 64 |
| 50      | 128   | 50÷128=0.39→0→128×0 | 0 |
| 200     | 32    | 200÷32=6.25→6→32×6 | 192 |
| 75      | 16    | 75÷16=4.68→4→16×4 | 64 |
| 1       | 4     | 1÷4=0.25→0→4×0 | 0 |

### 4.2 关键数字表（必须背下来！）

| 掩码最后段       | 256-值    | 2 的幂     | 主机位      | CIDR    | 块大小      |
| ----------- | -------- | -------- | -------- | ------- | -------- |
| 0           | 256      | 2⁸       | 8        | /24     | 256      |
| 128         | 128      | 2⁷       | 7        | /25     | 128      |
| 192         | 64       | 2⁶       | 6        | /26     | 64       |
| 224         | 32       | 2⁵       | 5        | /27     | 32       |
| 240         | 16       | 2⁴       | 4        | /28     | 16       |
| 248         | 8        | 2³       | 3        | /29     | 8        |
| 252         | 4        | 2²       | 2        | /30     | 4        |
| 254         | 2        | 2¹       | 1        | /31     | 2        |
| 255         | 1        | 2⁰       | 0        | /32     | 1        |

**记忆技巧**⭐:

```
【掩码序列】
128, 192, 224, 240, 248, 252, 254, 255
（每次增加：64, 32, 16, 8, 4, 2, 1）

【块大小序列】
128, 64, 32, 16, 8, 4, 2, 1
（每次除以 2）
```

## 第五部分：实战应用 💪

### 5.1 基础题型

**题目 1**: `192.168.1.100/26`

```
步骤：
1. 掩码最后段：192
2. 块大小：256 - 192 = 64
3. 网络地址：64 的倍数且≤100 → 64
4. 广播地址：64 + 64 - 1 = 127
5. 可用范围：192.168.1.65 ~ 192.168.1.126
6. 主机数：64 - 2 = 62

答案：
- 网络地址：192.168.1.64
- 广播地址：192.168.1.127
- 可用范围：192.168.1.65 ~ 192.168.1.126

```

4. 广播地址：0 + 128 - 1 = 127
5. 可用范围：10.0.0.1 ~ 10.0.0.126
6. 主机数：128 - 2 = 126

答案：
- 网络地址：10.0.0.0

## 第六部分：案例分析 📋

### 案例 1：公司网络规划

**需求**: 某公司有 4 个部门，每个部门最多 50 台电脑，使用 `192.168.1.0/24` 网段，如何划分子网？

```
分析：
- 每个部门 50 台 → 需要至少 52 个 IP（50+ 网络地址 + 广播地址）
- 块大小选择：64（因为 32 不够，64 够用）
- 掩码：256 - 64 = 192 → /26

划分方案：
部门 1: 192.168.1.0/26   (1-62)
部门 2: 192.168.1.64/26  (65-126)
部门 3: 192.168.1.128/26 (129-190)
部门 4: 192.168.1.192/26 (193-254)

每个子网：
- 网络地址：X.X.X.0/64/128/192
- 广播地址：X.X.X.63/127/191/255
- 可用主机：62 台（满足 50 台需求）
```

### 案例 2：VLSM 可变长子网掩码

**需求**: 使用 `192.168.1.0/24` 规划：
- 研发部：100 台
- 市场部：50 台
- 财务部：25 台
- 行政部：10 台

```
按从大到小分配：

1. 研发部（100 台）：
   - 需要块大小：128
   - 掩码：/25
   - 分配：192.168.1.0/25 (1-126)

2. 市场部（50 台）：
   - 需要块大小：64
   - 掩码：/26
   - 分配：192.168.1.128/26 (129-190)

3. 财务部（25 台）：
   - 需要块大小：32
   - 掩码：/27
   - 分配：192.168.1.192/27 (193-222)

4. 行政部（10 台）：
   - 需要块大小：16
   - 掩码：/28
   - 分配：192.168.1.224/28 (225-238)

剩余：192.168.1.240/28 (241-254) 备用
```

## 总结 🎓

### 核心要点

1. **记住公式**: `块大小 = 256 - 掩码最后段`
2. **背下序列**: 128, 192, 224, 240, 248, 252, 254, 255
3. **多练习**: 每天练 3 题，一周就能口算

### 速算口诀

```
二五六减掩码，得到块大小。
倍数找网络，加一第一个。
减一得广播，再减是末尾。
块大减二个，就是主机数。
```
