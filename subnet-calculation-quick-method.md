## 第二部分：前言 ⭐

### 为什么需要速算？

在实际工作中，我们经常遇到这样的场景：

| 场景 | 传统方法 | 块计算方法 |
|------|---------|-----------|
| **面试现场** | 掏出纸笔算半天 | 30 秒口算出答案 ✅ |
| **会议讨论** | "等我算一下..." | 脱口而出，专业感拉满 ✅ |
| **故障排查** | 打开计算器慢慢算 | 心算定位问题 ✅ |
| **方案设计** | 查表 + 验证 | 快速规划子网 ✅ |

**结论**: 块计算方法 > 二进制转换 > 查表法

### 什么是块计算方法？

**核心思想**: 把复杂的二进制运算，简化为简单的减法运算！

```
【传统方法】
IP: 192.168.1.100
掩码：255.255.255.192
→ 转二进制 → AND 运算 → 转十进制 → 结果
（耗时：2-3 分钟，容易出错）

【块计算方法】
掩码最后段：192
块大小：256 - 192 = 64
网络地址：64 的倍数且≤100 → 64
→ 结果：192.168.1.64
（耗时：10-15 秒，准确率高）
```

**重点核心**⭐: 记住 `256 - 掩码 = 块大小`，其他都是浮云！

## 第三部分：基础概念 📚

### 3.1 子网掩码是什么？

子网掩码（Subnet Mask）用于区分 IP 地址中的**网络部分**和**主机部分**。

```
IP 地址：   192.168.1.100
子网掩码：  255.255.255.0
           ─────┬──── ────┬───
           网络部分   主机部分
```

**通俗理解**: 
- 掩码 255 的段 → 网络部分（保持不变）
- 掩码 0 的段 → 主机部分（可以变化）

### 3.2 CIDR 表示法

CIDR（无类别域间路由）用 `/数字` 表示网络位数。

```
192.168.1.100/24
                ┬
                网络位数（前 24 位是网络部分）
```

**常见 CIDR**:

| 写法 | 含义 | 子网掩码 |
|------|------|---------|
| /8 | 前 8 位是网络位 | 255.0.0.0 |
| /16 | 前 16 位是网络位 | 255.255.0.0 |
| /24 | 前 24 位是网络位 | 255.255.255.0 |
| /26 | 前 26 位是网络位 | 255.255.255.192 |

### 3.3 关键概念速记

| 概念 | 定义 | 重要性 |
|------|------|--------|
| **网络地址** | 主机位全为 0 的地址 | ⭐⭐⭐ 子网的标识 |
| **广播地址** | 主机位全为 1 的地址 | ⭐⭐⭐ 子网内广播 |
| **可用主机** | 网络地址和广播地址之间的地址 | ⭐⭐⭐ 实际分配给设备 |
| **块大小** | 256 - 掩码最后段 | ⭐⭐⭐⭐⭐ 速算核心 |

## 第四部分：理论知识 🔬

### 4.1 块计算法核心公式

```
【核心公式】
块大小 = 256 - 掩码最后段

【推导公式】
网络地址 = 块大小的倍数且≤IP 最后段
广播地址 = 网络地址 + 块大小 - 1
第一个可用 = 网络地址 + 1
最后一个可用 = 广播地址 - 1
可用主机数 = 块大小 - 2
```

### 4.2 关键数字表（必须背下来！）

| 掩码最后段 | 256-值 | 2 的幂 | 主机位 | CIDR | 块大小 |
|-----------|--------|-------|--------|------|--------|
| 0 | 256 | 2⁸ | 8 | /24 | 256 |
| 128 | 128 | 2⁷ | 7 | /25 | 128 |
| 192 | 64 | 2⁶ | 6 | /26 | 64 |
| 224 | 32 | 2⁵ | 5 | /27 | 32 |
| 240 | 16 | 2⁴ | 4 | /28 | 16 |
| 248 | 8 | 2³ | 3 | /29 | 8 |
| 252 | 4 | 2² | 2 | /30 | 4 |
| 254 | 2 | 2¹ | 1 | /31 | 2 |
| 255 | 1 | 2⁰ | 0 | /32 | 1 |

**记忆技巧**⭐:

```
【掩码序列】
128, 192, 224, 240, 248, 252, 254, 255
 2⁷   2⁶   2⁵   2⁴   2³   2²   2¹   2⁰

【块大小序列】
128, 64, 32, 16, 8, 4, 2, 1
（每次减半，很好记！）

【CIDR 序列】
/25, /26, /27, /28, /29, /30, /31, /32
（每次 +1）
```

### 4.3 二进制规律

```
128 = 10000000  (1 个 1)
192 = 11000000  (2 个 1)
224 = 11100000  (3 个 1)
240 = 11110000  (4 个 1)
248 = 11111000  (5 个 1)
252 = 11111100  (6 个 1)
254 = 11111110  (7 个 1)
255 = 11111111  (8 个 1)
```

**规律**: 从左到右连续的 1，1 的个数 = 网络位数（在第 4 段中）

## 第五部分：实战应用 🛠️

### 5.1 场景一：CIDR 转掩码（口算）

**题目**: `/26` 对应的子网掩码是多少？

**传统方法** ❌:
```
26 = 8 + 8 + 8 + 2
第 4 段有 2 个网络位 → 6 个主机位
2⁶ = 64
256 - 64 = 192
结果：255.255.255.192
（耗时：30 秒）
```

**块计算方法** ✅:
```
直接背表！
/26 → 192
结果：255.255.255.192
（耗时：3 秒）
```

### 5.2 场景二：掩码转 CIDR（口算）

**题目**: `255.255.255.224` 对应的 CIDR 是多少？

**块计算方法** ✅:
```
最后段：224
256 - 224 = 32 = 2⁵
主机位 = 5 位
网络位 = 8 - 5 = 3 位（在第 4 段）
CIDR = 24 + 3 = /27
（耗时：5 秒）
```

### 5.3 场景三：计算网络地址（核心！）

**题目**: `192.168.1.100/26` 的网络地址是多少？

**块计算方法** ✅:
```
步骤 1: 确定块大小
/26 → 掩码 192 → 块大小 = 256 - 192 = 64

步骤 2: 找倍数
64 的倍数：0, 64, 128, 192, 256
100 在哪个区间？64-127 之间

步骤 3: 确定网络地址
网络地址 = 64（区间的起始值）

结果：192.168.1.64
（耗时：10 秒）
```

**图示**:
```
0    64    128   192   256
|-----|-----|-----|-----|
      ↑
     100 在这里
     所以网络地址是 64
```

### 5.4 场景四：计算广播地址

**题目**: `192.168.1.100/26` 的广播地址是多少？

**块计算方法** ✅:
```
已知：
- 网络地址 = 192.168.1.64
- 块大小 = 64

公式：广播地址 = 网络地址 + 块大小 - 1
计算：64 + 64 - 1 = 127

结果：192.168.1.127
（耗时：5 秒）
```

### 5.5 场景五：计算可用主机范围

**题目**: `192.168.1.100/26` 的可用主机范围是多少？

**块计算方法** ✅:
```
已知：
- 网络地址 = 192.168.1.64
- 广播地址 = 192.168.1.127

公式：
- 第一个可用 = 网络地址 + 1 = 65
- 最后一个可用 = 广播地址 - 1 = 126

结果：192.168.1.65 - 192.168.1.126
（耗时：5 秒）
```

### 5.6 场景六：计算可用主机数

**题目**: `/26` 能容纳多少台主机？

**块计算方法** ✅:
```
方法一：块大小 - 2
块大小 = 64
可用主机 = 64 - 2 = 62

方法二：2^主机位 - 2
主机位 = 6（从关键数字表查）
2⁶ - 2 = 64 - 2 = 62

结果：62 台
（耗时：3 秒）
```

### 案例 1: 基础计算（面试高频题）⭐

**题目**: IP: `172.16.50.100/27`

2. 网络地址
3. 广播地址
4. 可用主机范围
5. 可用主机数

**解答**:

```
【步骤 1: 子网掩码】
/27 → 查表 → 224
结果：255.255.255.224

【步骤 2: 网络地址】
块大小 = 256 - 224 = 32
32 的倍数：0, 32, 64, 96, 128...
100 在 96-127 之间
结果：172.16.50.96

【步骤 3: 广播地址】
96 + 32 - 1 = 127
结果：172.16.50.127

【步骤 4: 可用范围】
第一个可用：96 + 1 = 97
最后一个可用：127 - 1 = 126
结果：172.16.50.97 - 172.16.50.126

【步骤 5: 可用主机数】
块大小 - 2 = 32 - 2 = 30
结果：30 台

【总耗时】约 30 秒 ✅
```

### 案例 2: 子网划分（方案设计）⭐⭐

**题目**: 将 `192.168.1.0/24` 划分为 4 个子网

**解答**:

```
【步骤 1: 计算需要借几位】
4 个子网 = 2² → 需要借 2 位

【步骤 2: 计算新 CIDR】
原 CIDR: /24
借位：+2
新 CIDR: /26

【步骤 3: 确定新掩码】
/26 → 255.255.255.192

【步骤 4: 计算块大小】
256 - 192 = 64

【步骤 5: 列出所有子网】
子网 1: 192.168.1.0/26   (0-63)
子网 2: 192.168.1.64/26  (64-127)
子网 3: 192.168.1.128/26 (128-191)
子网 4: 192.168.1.192/26 (192-255)

【验证】
每个子网 62 台主机 × 4 = 248 台
原/24 有 254 台主机
254 - 248 = 6 台（损耗在子网边界）✅
```

### 案例 3: 根据主机数划分子网（实战）⭐⭐⭐

**题目**: `192.168.1.0/24`，需要容纳 50 台主机，怎么划分子网？

**解答**:

```
【步骤 1: 计算需要的主机位】
2^n - 2 ≥ 50
2⁶ - 2 = 62 ≥ 50 ✓
需要 6 位主机位

【步骤 2: 计算网络位】
32 - 6 = 26 位网络位
CIDR: /26

【步骤 3: 确定掩码】
/26 → 255.255.255.192

【步骤 4: 验证】
块大小 = 64
可用主机 = 64 - 2 = 62 台 ≥ 50 台 ✓

【结果】
使用 /26 子网掩码，每个子网可容纳 62 台主机
```

### 案例 4: VLSM 可变长子网掩码（高级）⭐⭐⭐⭐

**题目**: `192.168.1.0/24`，需要以下子网：
- 子网 A: 100 台主机
- 子网 B: 50 台主机
- 子网 C: 25 台主机
- 子网 D: 10 台主机

**解答**:

```
【原则】从大到小分配，避免浪费

【子网 A: 100 台】
2⁷ - 2 = 126 ≥ 100 ✓
主机位 = 7，网络位 = 25
CIDR: /25，块大小 = 128
范围：192.168.1.0/25 (0-127)

【子网 B: 50 台】
2⁶ - 2 = 62 ≥ 50 ✓
主机位 = 6，网络位 = 26
CIDR: /26，块大小 = 64
范围：192.168.1.128/26 (128-191)

【子网 C: 25 台】
2⁵ - 2 = 30 ≥ 25 ✓
主机位 = 5，网络位 = 27
CIDR: /27，块大小 = 32
范围：192.168.1.192/27 (192-223)

【子网 D: 10 台】
2⁴ - 2 = 14 ≥ 10 ✓
主机位 = 4，网络位 = 28
CIDR: /28，块大小 = 16
范围：192.168.1.224/28 (224-239)

【剩余备用】
192.168.1.240/28 (240-255)

【验证】
0-127:   子网 A (128 个地址)
128-191: 子网 B (64 个地址)
192-223: 子网 C (32 个地址)
224-239: 子网 D (16 个地址)
240-255: 备用 (16 个地址)
总计：256 个地址 = /24 ✅
```

### 案例 5: 跨段计算（易错点）⭐⭐⭐

**题目**: `10.50.100.200/12` 的网络地址是多少？

**解答**:

```
【注意】这不是/24 以上，不能直接用最后段！

【步骤 1: 分析 CIDR】
/12 = 8 + 4
第 1 段：8 位（全网络位）
第 2 段：4 位（部分网络位）

【步骤 2: 计算第 2 段的块大小】
第 2 段网络位 = 4 位
第 2 段主机位 = 8 - 4 = 4 位
块大小 = 2⁴ = 16

【步骤 3: 找倍数】
16 的倍数：0, 16, 32, 48, 64...
50 在 48-63 之间
第 2 段网络地址 = 48

【步骤 4: 组合结果】
第 1 段：10（不变）
第 2 段：48
第 3-4 段：0（主机位全 0）

结果：10.48.0.0/12
```

### 公式速记

```
【核心公式】
块大小 = 256 - 掩码最后段

【推导公式】
网络地址 = 块大小的倍数且≤IP 最后段
广播地址 = 网络地址 + 块大小 - 1
第一个可用 = 网络地址 + 1
最后一个可用 = 广播地址 - 1
可用主机数 = 块大小 - 2

【CIDR 计算】
CIDR = 24 + (8 - 主机位)  （对于/24 以上）
主机位 = log₂(块大小)
```

## 附录 C：错误代码速查 ⚠️

### 常见错误及解决方案

| 错误 | 原因 | 解决方案 |
|------|------|---------|
| 设备无法通信 | 不在同一子网 | 检查网络地址是否相同 |
| 网关无法访问 | 网关地址配置错误 | 确保网关在可用地址范围内 |
| IP 冲突 | 重复分配 IP | 使用 IP 地址管理表 |
| 子网重叠 | VLSM 规划不当 | 从大到小分配，避免重叠 |
| 浪费地址 | 子网过大 | 根据实际需求选择合适 CIDR |

### 易错点提醒

```
【易错点 1】混淆网络位和主机位
❌ 错误：/26 表示 26 个主机位
✅ 正确：/26 表示 26 个网络位，6 个主机位

【易错点 2】忘记减 2
❌ 错误：/26 有 2⁶ = 64 个可用主机
✅ 正确：/26 有 2⁶ - 2 = 62 个可用主机

【易错点 3】块大小计算错误
❌ 错误：/26 的块大小是 192
✅ 正确：/26 的块大小是 256 - 192 = 64

【易错点 4】使用网络地址或广播地址
❌ 错误：网关配置为 192.168.1.0（网络地址）
✅ 正确：网关配置为 192.168.1.1（第一个可用地址）

【易错点 5】跨段计算直接用最后段
❌ 错误：10.50.100.200/12 直接用 200 计算
✅ 正确：/12 在第 2 段，用 50 计算
```

网络地址：主机位全 0

网络地址 = 块大小的倍数且≤IP
广播地址 = 网络地址 + 块大小 - 1
可用主机数 = 块大小 - 2

【四个步骤】
1. 确定块大小
2. 找网络地址
3. 算广播地址
